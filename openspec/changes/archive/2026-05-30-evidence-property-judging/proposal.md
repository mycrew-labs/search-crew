## Why

实测的 deep/wide 综合报告"信息量太少、不够有逻辑和说服力"——平铺的 bullet list，缺推理链、缺多源印证的量化、缺分歧处理。

参考 OpenDeepSearch（sentient-agi/OpenDeepSearch）：它有「源可靠性分级 + 冲突显式处理 + 多源印证」的思路值得借鉴，但它的「域名硬分级（直答框>Wikipedia>.gov）」是天真的——Wikipedia 有粗制滥造条目，中英文互联网信息质量分布也不同。**用来源身份贴信任分，本质是用「它是谁」代替「它说得对不对」。**

本 change 用更本质的**证据属性评判**取代来源身份分级，并强制综合阶段保留异常值（反信息茧房）。

## What Changes

### evidence-search（采集时标注证据属性）
每条结果的 ranking 旁，新增三个属性标注：
- **证据类型**：客观规格 / 独立测评 / 用户体验 / 厂商宣传 / 二手转述
- **一手 vs 二手**：能否追到原始数据/官方规格/原始实测
- **lane**：中文源 / 英文源（已有，强调保留）

evidence-summary.md 的来源表加这几列，供综合阶段按属性论证。

### deep/wide 综合阶段（按证据属性论证 + 反茧房）
综合阶段 prompt 加「报告质量标准」：
1. **证据类型决定可信度**（领域相关，无通用权威源）：主张-来源不匹配则降级（厂商自夸性能、博客报参数 → 标利益相关/需第三方佐证）
2. **独立印证量化**：每个关键结论标「独立印证数（中 X 源 / 英 Y 源）」；去重——多篇引同一新闻稿算 1 源
3. **一手优先**：能追一手的标「已复核」，只有二手/单源的标「存疑」
4. **冲突显式裁决**：源分歧时分析原因（口径差异 / 地域差异），按证据属性给倾向，而非和稀泥或简单并列
5. **场景化推荐**：每个场景推荐附「核心支撑数据」
6. **反信息茧房**：MUST 单独保留「⚡ 异常 / 少数派 / 反直觉信号」段——呈现与主流共识不同的来源及其理由，不只报符合预期的结论

## Capabilities

### Modified Capabilities

- `evidence-search`：采集产物（evidence-search-NNN.md front-matter + evidence-summary.md 来源表）新增「证据类型 / 一手二手 / lane」标注。
- `deep-search`：综合阶段按证据属性论证 + 强制「异常/少数派」段。
- `wide-search`：同上（矩阵格里标证据类型 + 一手二手）。

## Impact

- 报告从"平铺 bullet"升级为"带证据强度的论证"，呼应 ODS 可靠性思路但判证据本身、不判出身。
- 反茧房段是元原则「用异常值防局部最优」的第一步落地（见记忆 evidence-evolving）。
- 纯 prompt / 产物模板改动，不改脚本代码。
- 这套标准是当前版本，预期随用户反馈迭代——非终态。
